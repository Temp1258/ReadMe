# Privacy Policy for ReadMe

**Last updated: March 27, 2026**

## Overview

ReadMe is a browser extension that captures audio from your browser tab or microphone and transcribes it to text using third-party speech-to-text services.

## Data Collection

ReadMe does **not** collect, store, or share any personal information. Specifically:

- No account registration is required
- No personal data is collected or transmitted to our servers
- No browsing history or user activity is tracked
- No cookies are used for tracking purposes

## Data Processing

When you use ReadMe to transcribe audio:

- Audio data from your active tab or microphone is sent directly to the speech-to-text provider you have configured (e.g., Deepgram)
- If AI summarization is enabled, transcribed text may be sent to OpenAI for processing
- All API calls are made directly from your browser to these services using your own API keys
- No audio or text data passes through any servers operated by ReadMe

## Data Storage

- Your settings and API keys are stored locally in your browser using Chrome's storage API
- Transcription results are stored locally in your browser
- No data is stored on external servers by ReadMe

## Third-Party Services

ReadMe may interact with the following third-party services based on your configuration:

- **Deepgram** (api.deepgram.com) — for speech-to-text transcription
- **OpenAI** (api.openai.com) — for AI-powered summarization
- **SiliconFlow** (api.siliconflow.cn) — for speech-to-text transcription (optimized for Chinese audio)

These services have their own privacy policies. Please review them for details on how they handle data.

## Changes

We may update this Privacy Policy from time to time. Any changes will be reflected in this document.

## Contact

If you have questions about this Privacy Policy, please open an issue on our GitHub repository.
